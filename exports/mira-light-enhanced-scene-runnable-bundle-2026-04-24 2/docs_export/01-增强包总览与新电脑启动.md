# 增强包总览与新电脑启动

这份文档说明这个增强包到底是什么，以及在新电脑上应该先怎么启动。

## 一句话定位

这是一个：

`可直接运行的 release 底座 + 主仓场景源码镜像 + 主控台补充材料`

组合包。

## 两层内容

### 1. release 可运行层

这部分沿用了原始 `mira-light-release-runnable-bundle` 的结构，适合在新电脑上直接启动：

- `scripts/`
- `web/`
- `tools/mira_light_bridge/`
- `assets/`
- `fixtures/`
- `docs/`

### 2. 主仓源码镜像层

这部分用于阅读、对照和继续开发：

- `main_repo_source_copy/scripts/`
- `main_repo_source_copy/web/`
- `main_repo_source_copy/Motions/`
- `main_repo_source_copy/tools/mira_light_bridge/`

## 新电脑最短启动路径

### 基础安装

```bash
bash scripts/one_click_install.sh
```

### 启动完整本地栈

```bash
bash scripts/start_local_stack.sh
```

### 浏览器打开导演台

```text
http://127.0.0.1:8765/
```

## 可选增强：带 motion scripts 的导演台

这个增强包另外附带：

- `Motions/01_wake_up/scene_script.py`
- ...
- `Motions/10_sleep/scene_script.py`
- `scripts/console_server_with_motion_scripts.py`
- `scripts/mira_light_signal_delivery.py`

如果你希望在新电脑上直接看到主仓那套 per-scene motion script 组织方式，可以运行：

```bash
bash scripts/start_director_console_with_motion_scripts.sh
```

## 为什么不直接覆盖 release 底座

原因很简单：

- release 版更适合“新电脑先跑起来”
- 主仓源码更适合“继续开发和对照”

所以这个包刻意保留双层结构，避免把“稳定交付入口”和“当前主仓源码镜像”混成一个东西。
