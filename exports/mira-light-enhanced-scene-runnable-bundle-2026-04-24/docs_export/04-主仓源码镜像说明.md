# 主仓源码镜像说明

这份文档说明增强包里 `main_repo_source_copy/` 这一层到底放了什么，以及它和 runnable 底座是什么关系。

## 设计原则

我没有直接把主仓文件覆盖 release 可运行底座，而是做成：

```text
可运行底座
+ 主仓源码镜像
```

这样做的原因是：

- 你在新电脑上可以先跑起来
- 你又不会丢失主仓当前那套 scene / console / motion scripts 组织方式

你可以直接对照：

- `scripts/`

- 哪些是 release 稳定版
