# Memory Notes

Use this folder for two kinds of files:

- durable operating notes that should outlive a single session
- dated rollout or debugging logs such as `2026-04-09-machine-a.md`

Recommended split:

- `MEMORY.md`
  - stable truths about Mira, the machine, scene policy, and safety
- `memory/YYYY-MM-DD-*.md`
  - daily rollout notes, route changes, calibration results, service incidents,
    and hardware observations

Only move facts into `MEMORY.md` when they should persist.
