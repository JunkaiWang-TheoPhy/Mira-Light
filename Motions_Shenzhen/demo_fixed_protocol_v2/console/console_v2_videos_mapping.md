## 最容易混淆的三件事

### 1. `console v2 04` 不是 `Videos/04`

- `console v2 04`
  - 是 `04_offer_celebrate`
  - 对应 `Videos/05`
- `Videos/04`
  - 被并入 `03_hand_nuzzle`
  - 作为第 3 个动作的短版目标接近 fallback

### 2. PDF 第 4 条不是 console 里的 `04`

- PDF 第 `4` 条：追踪 / 展示感知能力
- 在 `console v2` 里对应的是 `07_tabletop_follow`
- 对应视频是 `Videos/08`

### 3. `Videos/03` 和 `Videos/04` 都归到 console 的 `03`

当前 `console v2` 设计里：

- `Videos/03`
  - 完整的 hand nuzzle / touch affection，是默认主参考
- `Videos/04`
  - 偏短版目标接近 / 桌边接触片段，只作 fallback

但它们都服务于同一个场景：

- `03_hand_nuzzle`

默认由 `03_hand_nuzzle_faithful_demo.py` 执行；必要时旧脚本 `03_hand_nuzzle_demo.py --variant 04` 可以临时替代。

## 按现场操作顺序理解

如果按一轮完整演示来理解，`console v2` 的顺序其实是：

1. `01` 起床
2. `02` 认人试探
3. `03` 摸一摸 / 亲近
4. `04` 好消息庆祝
5. `05` 送别
6. `06` 睡觉
7. `07` 桌面追踪
8. `08` 拍照 / 定格姿态

所以它不是严格按 PDF 原始编号排的，而是按当前 demo console 的演示逻辑重排过的。

## 一句话总结

如果你以后只想快速查：

- `console v2 04`
  - 看 `Videos/05`
  - 做“offer 庆祝 / 跳舞模式”
- `Videos/04`
  - 不单独占一个 console 编号
  - 它并入 `console v2 03`
