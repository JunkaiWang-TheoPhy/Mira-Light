## 当前版本的总体策略

第一版先做 **固定动作版**。

原因：

- 当前端侧没有一条成熟的“桌面目标 -> 舵机持续更新”板端脚本
- 这轮 `4 5 6` 又不是主推进项
- 如果现在硬做成实时 tracking，很容易做得既不稳也不好讲

所以第一版应该是：

- 一条诚实的说明性 fixed script
- 先把节奏和桌面感做对

## 推荐脚本名

建议脚本名直接叫：

```text
sz_tabletop_follow_fixed.py
```

如果先只落到 repo 的 scene 层，则未来可对应：

```text
scripts/scenes_shenzhen.py -> sz_tabletop_follow
```

## 推荐动作分段

建议拆成 **5 段**。

### 第 0 段：进入桌面关注姿态

目标：

- 把灯头压低一点
- 明确告诉观众它现在在看桌面，不是在看人脸

这一段必须有。

不然观众很难理解后面的左右移动是在“跟桌上的书”。

建议：

- 切到桌面工作位
- 灯光改为功能性白光，而不是暖情绪光

## 第 1 段：看向桌面左侧

目标：

- 建立第一个明确目标点
- 让观众理解“书在左边”

动作要求：

- 慢慢过去
- 到位后停 `0.5s ~ 0.8s`

## 第 2 段：平滑跟到中间

目标：

- 表达“它在跟”
- 不是“它会摆头”

这一段最重要的是：

- 平滑
- 不抖
- 不突兀

建议时长：

- `0.8s ~ 1.2s`

## 第 3 段：继续到右侧并停住

目标：

- 完成 “左 -> 中 -> 右” 的主证据链
- 在右侧停住，证明“目标停，它也停”

这一段是整条 fixed script 的证据高点。

## 第 4 段：再跟一下

目标：

- 对应 PDF 里“评委再移动，Mira 再跟”
- 第一版不需要真输入
- 只做一小段再次跟随的占位动作

这一步不能太大。

它应该像：

- 目标又挪了一点
- Mira 补一个短跟随

而不是重新来一遍完整左中右。

## 第 5 段：回到桌面等待位

目标：

- 结束后仍保持“桌面工作位”
- 不要直接抬头回到看人位

这样后续如果接别的桌面互动，会更自然。

    ...

def follow_to_right():

    ...

    ...

## 第一版为什么不能假装是真 tracking

因为 PDF 这条真正想证明的是“感知能力”。

如果 fixed 版在表达上不诚实，观众很容易一试就知道它不是 tracking。

所以第一版必须明确定位为：

- 说明性 fallback
- 节奏验证版
- 舞台占位版

## 当前固定版能证明什么

它能证明：

- Mira 的注意力可以沿桌面移动
- 灯头和灯光可以一起形成“桌面追踪感”
- 观众能理解它是在看桌面目标

它还不能证明：

- Mira 真正锁住了某本书
- Mira 能在有干扰物时稳定不乱切
- Mira 具有真实持续 tracking 闭环

## 第二版再做什么

第二版再升级：

1. `targetId`
2. `horizontalZone`
3. `lockStrength`
4. 真正的“停下就停、再动再跟”

也就是说：

- 第一版：固定左中右
- 第二版：事件驱动分段
- 第三版：实时 tracking

## 一句话总结

如果按 `Videos/04` 和这个 PDF 副本第 4 条来设计，第一版最正确的做法不是实时追踪，而是：

> 先做一条固定的桌面跟随证据脚本：进入桌面位 -> 左 -> 中 -> 右 -> 停住 -> 再跟一下 -> 回桌面等待位。
