# `sz_cautious_intro` 具体脚本蓝图

这份文档是在 [02_sz_cautious_intro_fixed.md](./02_sz_cautious_intro_fixed.md) 的基础上，再往前推进一层。

它专门回答：

> 如果基于 `02_20260425_043823_01.mp4` 和 `Mira Light 展位交互方案 副本.pdf` 第二条来写第一版脚本，应该怎样具体拆动作。

## 参考依据

### 视频

- `docs/Shenzhen/Videos/02_20260425_043823_01.mp4`

这段视频时长约 `133.7s`，明显比第一条更长，说明它更像：

- 一段多阶段互动
- 而不是单一“大动作”

所以在脚本设计上，第二条不能写成“一个大姿态 + 一个收尾”。

它更适合拆成：

- 试探
- 靠近
- 回避
- 再回看

这种有心理层次的链。

### PDF 第二条

根据 `Mira Light 展位交互方案 副本.pdf` 第 `3~6` 页，第二条“好奇你是谁”的关键点是：

1. 灯头看向评委
2. 底座往用户方向靠
3. 缓慢摇头一次
4. 更靠近
5. 转向远离用户
6. 低头并上下灯头，表示害羞
7. 再转向你，探出来，左右移动
8. 面向你点头
9. 如果评委伸手或靠近，往远离评委侧转并下沉
10. 害羞结束后，再慢慢往回和往前靠

这说明第二条的核心不是“好奇地看一眼”，而是：

> Mira 先试探确认你，再因为紧张退开，然后又忍不住回来。

## 当前版本的总体策略

第一版先做 **固定方向版**，不要急着接实时识别。

原因：

- 第二条的重点是心理节奏
- 不是 tracking
- 如果一开始就接不稳定方向输入，很容易丢掉“犹豫”的表演层次

所以第一版建议：

- 固定一个“评委在右前方”的版本
- 或固定一个“主持人常站位”的版本

## 推荐脚本名

建议脚本名直接叫：

```text
sz_cautious_intro_fixed.py
```

如果先只落到 repo 的 scene 层，则未来可对应：


## 推荐动作分段

建议拆成 **7 段**。

### 第 0 段：中性等待位

目标：

- 所有“试探认人”动作都从一个相对平静、可预测的起点开始

要求：

- 先不要用睡姿起步
- 这条是“互动中的认人”，不是“从睡眠中醒来”

建议起点：

- `normal_pose`
- 或者轻微朝主持人侧的等待位

## 第 1 段：先看向评委

目标：

- 表达“我注意到你了”
- 但还没有立刻前探

第一版建议：

- 固定朝右前方看过去
- 速度要慢
- 到位后停一下

可借板端材料：

- `servo_1_3_slow_1800_2750.py`
- 或自己定义一个固定 look pose

建议时长：

- `0.6s ~ 0.9s`

## 第 2 段：轻轻靠近

目标：

- 表达“我想确认一下你是谁”

这一段不要做得太深。

第一版建议直接借：

- `servo_1_2_lean_forward_2148_1848.py`

这条天然适合“试探性靠近”。

建议时长：

- `0.8s ~ 1.2s`

## 第 3 段：缓慢摇头一次

目标：

- 不是拒绝
- 而是“辨认 / 犹豫 / 不确定”

所以这个“摇头”一定要慢，而且次数只做一次。

可借板端材料：

- `servo_3_shake_2100_2000.py`

但不要全量照搬次数，建议在第一版里只做 **半幅或单轮**。

建议时长：

- `0.6s ~ 0.8s`

## 第 4 段：再更靠近一点

目标：

- 比第一段前探更深一点
- 给后面的“害羞退开”制造对比

实现方式：

- 再做一次前倾
- 但比第 2 段多一点停顿

这一步可以复用：

- 同一个 `lean_forward` 脚本
- 或在 scene 里单独定义 `lean_forward_soft` / `lean_forward_deeper`

## 第 5 段：害羞退开

目标：

- 这是整条 scene 的情绪转折点

动作要求：

- 转向远离评委的一侧
- 低头
- 做一小段上下点动，像害羞

可借板端材料：

- `servo_1_2_dodge_1848_1808.py`
- `servo_2_nod_1900_2200.py`

组合方式建议：

1. 先 `dodge`
2. 停半拍
3. 小幅 `nod`

建议时长：

- `1.0s ~ 1.4s`

## 第 6 段：再探回来，左右小幅移动

目标：

- 表达“还是忍不住想看看你”

这一步不要做成大幅扫视。

更好的做法是：

- 回到一个比起始更靠近用户一点的 pose
- 再加很小的左右偏移

可借板端材料：

- `servo_1_2_lean_forward_2148_1848.py`
- `servo_3_shake_2100_2000.py` 的节奏思路

但建议不要直接用完整 `shake`，只取“小左右摆一次”的概念。

## 第 7 段：面对你点头一下

目标：

- 给这条 scene 一个明确的“我决定和你建立联系”收尾

可借板端材料：

- `servo_2_nod_1900_2200.py`

第一版建议：

- 只做一次点头
- 不要做双点头

否则会从“认你”变成“打招呼”。

## 可选分支：如果对方继续逼近

PDF 里还有一步：

- 如果评委伸手或者继续靠近，它会往远离评委一侧转并下降

这一条第一版可以先不做实时分支。

建议先作为：

- 第二版 builder 才接入的补充分支

第一版只保留主链。

## 第一版脚本结构建议

建议写成这种结构：

```python
def look_to_guest():
    ...

def lean_forward_soft():
    ...

def shake_once_slow():
    ...

def lean_forward_deeper():
    ...

def shy_away():
    ...

def peek_back():
    ...

def nod_once():
    ...

def main():
    look_to_guest()
    lean_forward_soft()
    shake_once_slow()
    lean_forward_deeper()
    shy_away()
    peek_back()
    nod_once()
```

## 第一版最重要的限制

不要做成下面这些错误版本：

- 一上来就大幅看向用户
- 没有停顿，直接一串动作跑完
- 害羞阶段做得太剧烈，像逃跑
- 点头太多，变成过度讨好

## 当前固定版能证明什么

它能证明：

- Mira 的注意力有心理层次
- 它不是机械转头
- 它会试探、退缩、再回来

它还不能证明：

- Mira 真的知道眼前是不是主人
- Mira 真在根据实时人脸和距离输入做判断

## 第二版再做什么

等固定版稳定后，再升级：

1. 把第 1 段和第 5 段接 `targetDirection`
2. 把是否走“害羞分支”接 `ownerFaceFound`
3. 把最后的回看深度接 `targetDistanceBand`

## 一句话总结

第二条“好奇你是谁”的第一版最正确写法不是实时跟人，而是：

> 用固定方向、固定靠近、固定摇头、固定害羞退开和固定回看，先把“认人时的心理节奏”稳定演出来。
