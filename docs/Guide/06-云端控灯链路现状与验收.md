# 云端控灯链路现状与验收

## 1. 当前结论

现在的 live 语音和飞书入口，已经不是“只能聊天”。

当前 active runtime 已经具备：

- 云端 `OpenClaw main agent` 调用 `mira-light-bridge` 工具
- 通过反向隧道访问本机 bridge
- 触发本机 `Mira Light` runtime

但当前仍有一个边界：

- `deviceOnline=false`

所以现在可以说：

- 云端控灯工具链已打通

但还不能说：

- 真灯硬件动作验收已完成

## 2. 当前链路

```text
语音 / 飞书
-> 远端 active OpenClaw gateway
-> mira-light-bridge plugin
-> 远端 127.0.0.1:19783
-> SSH reverse tunnel
-> 本机 127.0.0.1:9783
-> 本机 mira-light-bridge
-> 本机 runtime
-> 小灯
```

## 3. 先看是不是桥接已通

检查本机 bridge：

```bash
curl -s http://127.0.0.1:9783/health
```

检查本机 runtime：

```bash
curl -s http://127.0.0.1:9783/v1/mira-light/runtime \
  -H 'Authorization: Bearer <BRIDGE_TOKEN>'
```

重点看：

```json
"sceneCount": 13
```

这通常表示：

- bridge 和 profile 正常

再看：

```json
"deviceOnline": true
```

如果这里还是 `false`，说明硬件还没上线。

## 4. 先用读工具验收

最稳的第一步，不是先跑动作，而是先让 live 语音链调用：

- `mira_light_runtime_status`

当前正确结果应类似：

```json
{"running":false,"deviceOnline":false,"sceneCount":13,"scene":null}
```

如果已经不再是：

```text
TOOL_UNAVAILABLE
```

就说明：

- active runtime 已经接上 `mira-light-bridge`

## 5. 再用写工具验收

当前推荐用真实存在、风险较低的 scene：

- `cute_probe`
- `farewell`
- `sleep`
- `wake_up`

当前不要用：

- `welcome`

因为它不是当前 profile 的 scene id。

## 6. 写动作时的推荐参数

当前推荐默认都走：

```json
"async": true
```

原因很直接：

- `async=false` 可能返回 `timed out`
- 但 scene 实际已经开始执行

所以 booth/live 场景下，`async=true` 更稳。

## 7. 这轮已经验证过什么

已经验证通过的包括：

- `mira_light_runtime_status`
- `mira_light_list_scenes`
- `mira_light_run_scene(scene=cute_probe, async=true, allowUnavailable=true)`

其中 `cute_probe` 的 live 写操作已经表现为：

- 云端返回 `ok=true`
- 本机 runtime 记录到 `lastStartedAt`
- 本机 runtime 记录到 `lastFinishedScene = cute_probe`

这说明写操作链已经通到本地 runtime。

## 8. 如何判断“只是桥通了”还是“真灯也在线了”

看这个字段：

```json
"deviceOnline": true
```

如果它是 `false`：

- 云端工具链可能已经是通的
- scene 触发逻辑也可能已经跑了
- 但还不能算硬件验收通过

如果它是 `true`：

- 再加上 scene 真有动作
- 才能算“真灯动作验收通过”

## 9. 下一步怎么验

推荐顺序：

1. 先让小灯上线
2. 再确认 `deviceOnline=true`
3. 再测 `cute_probe`
4. 再测 `farewell`
5. 最后再把云端动作策略接进实时语音主链

## 10. 一句话记忆

现在已经完成的是：

- 云端 active runtime 已能实际调用 `Mira Light` 工具

现在还没完成的是：

- 小灯硬件在线后的最终动作验收
