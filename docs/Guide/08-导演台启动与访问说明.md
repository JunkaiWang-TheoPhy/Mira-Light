# 导演台启动与访问说明

这篇说明专门回答两个现场最常见的问题：

- 导演台怎么启动
- 导演台对应的网址是什么
- 无声版导演台怎么打开

对应代码入口：

- [scripts/console_server.py](/Users/huhulitong/Documents/GitHub/Mira-Light/scripts/console_server.py)
- [web/index.html](/Users/huhulitong/Documents/GitHub/Mira-Light/web/index.html)
- [tools/mira_light_bridge/README.md](/Users/huhulitong/Documents/GitHub/Mira-Light/tools/mira_light_bridge/README.md)

## 1. 先记住默认地址

当前默认端口是：

```text
http://127.0.0.1:8765/
```

这是原版导演台网页的地址。

同一套 `console_server.py` 现在还会同时提供无声版页面：

```text
http://127.0.0.1:8765/silent.html
```

它背后默认代理到本地 bridge：

```text
http://127.0.0.1:9783
```

也就是说：

- `8765` 是导演台网页
- `9783` 是 bridge API

## 2. 最小启动方式

如果你本机 bridge 还没起，推荐按这个顺序启动。

先起 bridge：

```bash
export MIRA_LIGHT_BRIDGE_TOKEN=test-token
zsh tools/mira_light_bridge/start_bridge.sh
```

再起导演台：

```bash
python3 scripts/console_server.py
```

看到类似输出：

```text
[console] starting at http://127.0.0.1:8765
[console] proxying bridge http://127.0.0.1:9783
```

这时浏览器打开：

```text
http://127.0.0.1:8765/
```

如果需要无声版，则打开：

```text
http://127.0.0.1:8765/silent.html
```

## 3. 如果 bridge 已经在跑

如果你的 bridge 已经在本机默认端口上运行，那么只需要启动导演台：

```bash
python3 scripts/console_server.py
```

然后打开：

```text
http://127.0.0.1:8765/
```

或无声版：

```text
http://127.0.0.1:8765/silent.html
```

## 4. 如果 bridge 不在默认端口

例如你现在接的是反向隧道后的本地端口，或别的 bridge 地址，可以显式指定：

```bash
python3 scripts/console_server.py --bridge-base-url http://127.0.0.1:19783
```

这时网页地址仍然是：

```text
http://127.0.0.1:8765/
```

变化的是导演台背后代理的 bridge 地址，不是网页端口。

无声版地址仍然是：

```text
http://127.0.0.1:8765/silent.html
```

## 5. 如果想让局域网别的设备访问

把导演台绑定到 `0.0.0.0`：

```bash
python3 scripts/console_server.py --host 0.0.0.0 --port 8765
```

然后在局域网里访问：

```text
http://你的电脑IP:8765/
```

例如：

```text
http://192.168.31.23:8765/
```

无声版则是在后面加 `/silent.html`：

```text
http://192.168.31.23:8765/silent.html
```

## 6. 怎么快速判断是不是起成功了

### 先看网页服务

```bash
curl -s http://127.0.0.1:8765/
```

如果返回 HTML，就说明导演台网页服务已经起来了。

### 再看 bridge

```bash
curl -s http://127.0.0.1:9783/health
```

如果这里不通，常见现象是：

- 网页能打开
- 但状态区一直空着
- 点按钮会报错

因为当前网页是 `console_server.py` 托管静态资源，但运行状态、scene、vision、profile 都还要经过 bridge。

## 7. 原版和无声版的区别

- 原版导演台：`http://127.0.0.1:8765/`
- 无声版导演台：`http://127.0.0.1:8765/silent.html`
- 两个页面可以同时打开，因为它们现在使用独立的本地页面状态，不会互相覆盖 `cue mode` 和“附件 / 依赖就绪”的勾选。
- 但它们仍然控制同一个 bridge、runtime 和真灯，所以一边触发场景，另一边会同步看到运行状态变化。
- 无声版会固定使用 `scene` cue，并对场景 / trigger 请求带上 `silentMode=true`，这样 runtime 会跳过主持人口播和场景中的音频步骤。

## 8. 当前最常见的现场误区

### 误区 1：能打开网页就等于导演台全通

不是。

网页能打开，只能说明：

- `scripts/console_server.py` 正在运行

它不代表：

- bridge 已通
- runtime 已通
- 真灯在线

### 误区 2：改网页地址就能改控灯地址

不是。

当前页面里的 `Lamp Base URL` 是只读展示位，导演台真正代理的 bridge 地址来自：

- `scripts/console_server.py` 默认值
- 或 `--bridge-base-url`

## 9. 推荐的现场启动顺序

建议固定成下面这套：

1. 先确认 bridge 正常：`127.0.0.1:9783`
2. 再启动导演台：`python3 scripts/console_server.py`
3. 打开导演台：`http://127.0.0.1:8765/`
4. 如果现场需要安静演示，再额外打开：`http://127.0.0.1:8765/silent.html`
5. 先看状态条是否刷新
6. 再点 `refresh /status` 和 `refresh /led`
7. 最后再触发 scene 或 live trigger

## 10. 一句话记忆

当前导演台默认就是：

```text
原版网页：http://127.0.0.1:8765/
无声网页：http://127.0.0.1:8765/silent.html
bridge：http://127.0.0.1:9783
```

先起 bridge，再起 `console_server.py`，然后按需要打开原版或无声版页面。
